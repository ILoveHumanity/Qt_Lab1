## Постановка задачи
Реализовать защиту данных пользовательских папок и файлов, находящихся в папке, а также подпапках путем шифрования. 
Для доступа к данным исходной папке необходимо выполнить дешифрование.

### Алгоритм шифрования

Для шифрования используется схема на базе PBKDF2 + AES-256-GCM.

Шаги шифрования:
1. Генерируется случайная соль для получения ключа из пароля, размером *saltSize* байт.
2. Из пользовательского пароля и соли с помощью PBKDF2 на *iterationCount* итераций формируется 256-битный ключ.
3. Генерируется IV (вектор инициализации), обеспечивающее уникальность шифрования, для GCM, размером *IVSize* байт.
4. Файл шифруется алгоритмом AES-256-GCM.
5. Вычисляется и сохраняется контрольная сумма (tag), размером *tagSize* байт, которая обеспечивает контроль целостности и аутентификацию.

Зашифрованный файл состоит из:
- *EncryptedFlag* (Последовательность символов обозначающее зашифрованность файла)
- *salt* (Соль, использованная для получения ключа из пароля)
- *IV* (Вектор инициализации, использованный при шифровании этого файла)
- *Шифротекст*
- *tag* (контрольная сумма)

Шаги дешифрования:
1. Считывается все содержимое файла.
2. Проверяется размер файла и совпадение *EncryptedFlag*.
3. Выделяются *salt* , *IV*, *Шифротекст*, *Tag*.
4. По паролю и соли из файла получается ключ с помощью PBKDF2.
5. Выполняется дешифрование AES-256-GCM
6. Выполняется проверка проверяется *tag*


## Инструкция для пользователя
1) Соберите проект
2) При запуске можно указать флаг `<MODE>`
    - Если он не указан, программа попросит ввести его.
3) После этого программа попросит ввести путь до папки.
4) После этого программа попросит ввести пароль.
    - Ограничение на пароль: 20 символов.

Доступные варианты работы `<MODE>`: 
-   enc / encrypt   (Шифрование файлов)
-   dec / decrypt   (Дешифрование файлов)
-   ls  / list      (Вывод списка файлов)
